## Что делает этот промпт

1. **Читает план и статус** → знает где остановились
2. **Определяет следующий этап** автоматически
3. **Применяет все правила** (TodoWrite, /feature-dev, локальный code review перед push, vibe-coding осторожность, production-aware)
4. **Обновляет статус после завершения** → следующий чат знает с чего стартовать
5. **Один промпт на все этапы** → не надо подбирать какой использовать

## Когда менять чат

- После каждого этапа — новый чат с тем же промптом
- Если внутри этапа стало > 30 сообщений — `/clear` или новый чат с тем же промптом (Claude перечитает статус)
- Если контекст путается — новый чат

## Если что-то идёт не так

**Claude не нашёл план или статус**: проверь что ты в папке проекта (`/Users/mihailvoinskij/Yandex.Disk.localized/Projects/msgnn-calculator/`).

**Claude хочет проскочить code review перед push**: остановись, скажи «нет, сначала запусти `feature-dev:code-reviewer` через Agent tool». Это правило проекта, ревью Claude запускает САМ (это локальный subagent, не платный плагин).

**Claude делает не то что надо**: открой plan mode (`/plan` или Esc+P) и обсуди.

**Что-то непонятно в коде**: спроси прямо в чате «объясни что значит вот эта строка, простыми словами».
